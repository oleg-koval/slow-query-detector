## Design Context

### Users

Backend and full-stack Node.js developers evaluating or adopting a small OSS library for query latency visibility—often in dim editors or terminals, skimming docs quickly before `npm install`.

### Brand Personality

Instrument-like, calm, and exacting—closer to a lab readout than a startup landing page. Trust through restraint and readable structure, not decoration.

### Aesthetic Direction

Dark, softly tinted neutrals (cool graphite, not pure black), one disciplined accent (jade / sea-glass, not neon cyan), asymmetric left-led layout with a clear typographic hierarchy. Anti-reference: centered “slide deck” marketing blocks, raster logos with light boxes on dark UI, gradient-stuffed hero templates.

### Design Principles

1. **Alignment serves meaning** — prose and code start at the same logical margin; code is always left-aligned and horizontally scrollable.
2. **Surfaces are tinted, not pure** — backgrounds and text use OKLCH with controlled chroma; accents stay below ~12% of visual weight.
3. **Mark without raster hacks** — wordmark renders as text + vector pulse so GitHub Pages never shows a white logo slab.
4. **One motion story** — light stagger on load; respect `prefers-reduced-motion`.
5. **Accessible defaults** — visible focus, skip link, sufficient contrast on body and controls.
